AuraBot: Accessible Mood Tracker

By [Your Team Name]

Introduction

AuraBot is a Discord-based mood tracker designed for individuals with ADHD, anxiety, and related mental health challenges. It enables users to privately log their moods, visualize trends, and stay motivated with gamified features. Our goal is to create a tool that prioritizes accessibility, privacy, and user agency.

This project addresses the gap in accessible mood-tracking tools by ensuring compatibility with assistive technologies like screen readers and by emphasizing data security.

Positive Disability Principles

We applied Positive Disability Principles by avoiding ableist assumptions, centering user agency, and designing for inclusivity. Features such as customizable prompts and streak-based motivation empower users to track emotional well-being on their own terms.

  • Non-ableist language: Avoided stigmatizing terms and ensured descriptions were inclusive.
  • User control: Allowing users to delete entries and customize reminders.
  • Cross-disability solidarity: Considering diverse needs, including ADHD and anxiety.

Related Work

Existing mood-tracking apps like Daylio and Reflectly inspired our work. However, these tools often lack accessibility features, such as compatibility with screen readers or options for private logging on platforms like Discord. AuraBot addresses these gaps by integrating assistive technology support and prioritizing user privacy.

Methodology and Results

We developed AuraBot using Python and the Discord API, with MongoDB for secure data storage. The bot includes features like:

  • Private mood logging via Discord commands.
  • Customizable daily check-ins and reminders.
  • Data visualization for mood trends.

Our metrics for success included screen-reader compatibility, secure data encryption, and positive user feedback during testing.

Disability Justice Analysis

This project aligns with principles of disability justice, including:

  • Intersectionality: Considering overlapping identities, such as ADHD and anxiety, in design decisions.
  • Anti-capitalism: Providing a free tool without monetization pressures.
  • Cross-disability solidarity: Designing for multiple disabilities, including visual and cognitive impairments.

Learnings and Future Work

We learned the importance of prioritizing accessibility from the start. Future iterations could include multilingual support, more advanced data visualizations, and expanded integrations with other platforms.
